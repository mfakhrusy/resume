Project

GPU Orchestrator

Open-source GPU orchestration platform for Kubernetes. An alternative to NVIDIA Run:ai for managing GPU resources across multiple clusters.

Go React TypeScript PostgreSQL TimescaleDB Python Docker SSE JWT RBAC

Features

Multi-Tenant

Organization-level isolation with tenant admins and researchers.

Multi-Cluster Management

Connect and manage multiple GPU clusters from a single control plane.

GPU Quota Management

Per-project GPU allocation with enforcement.

Workload Templates

Pre-configured environments for ML training and inference.

Model Catalog

Curated inference models — Llama, Mistral, Stable Diffusion, and more.

Real-Time Metrics

Live GPU utilization, memory, and temperature via SSE.

Historical Metrics

Time-series storage with TimescaleDB for cost attribution.

CLI Tool

gpuorch command for workload submission and monitoring.

RBAC

3-tier role model: platform_admin, tenant_admin, researcher.

User Management

Admins can create, edit, deactivate, and delete users.

External Cluster Model

Control plane only — no embedded Kubernetes. Clusters run as separate services that the API server communicates with.

Python Simulator

Development mode uses a Python simulator that generates realistic GPU metrics without requiring real hardware.

JWT Auth with 3-Tier RBAC

Platform admins, tenant admins, and researchers — each with scoped permissions enforced at the API layer.

CLI Usage 

# Login
gpuorch login --server http://localhost:8080

# Set context
gpuorch tenant use acme
gpuorch cluster use my-cluster

# Submit a training job
gpuorch submit --name llama-finetune --project llm-finetuning \
  --template llm-finetuning --gpus 4

# Monitor
gpuorch ps --watch
gpuorch logs llama-finetune --follow
